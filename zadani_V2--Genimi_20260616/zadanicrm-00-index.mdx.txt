## 1. Vize a účel systému

FosterFlow není generické CRM. Jde o specializovaný "operační systém" pro agendu sociálně-právní ochrany dětí (SPOD). Systém pracuje se zvláštními kategoriemi osobních údajů (zdravotní zprávy, soudní rozhodnutí), eviduje rodiny, hlídá čerpání státních dotací na výkon pěstounské péče (SPVPP) a procesuje agendu soudů a úřadů. 

Cílem je radikálně minimalizovat administrativu pracovníků v terénu pomocí AI a maximalizovat auditní kontrolu a přehled pro vedení organizace.

## 3. Technologický stack

Tento technologický stack je závazný a musí být použit při inicializaci projektu.

### Frontend
* **Framework:** Next.js (App Router)
* **Knihovna:** React 18+
* **Styling:** Tailwind CSS (striktně využívat utility classes, bez vlastních CSS souborů, pokud to není nezbytně nutné).
* **Ikony:** Lucide React
* **PWA:** Implementace pomocí `next-pwa` nebo nativních Service Workers s využitím `IndexedDB` pro lokální offline ukládání transakcí.

### Backend & Databáze
* **Platforma:** Supabase
* **Databáze:** PostgreSQL 15+
* **Bezpečnost:** Row Level Security (RLS) aplikované na každou tabulku. Autorizace přes Supabase Auth.
* **Vektorová databáze:** Rozšíření `pgvector` pro Knowledge Base.

### Infrastruktura & Integrace (GCP)
* **Hosting Frontendu:** Webglobe (Node.js / Statický export dle možností tarifu).
* **Google Workspace API:** Google Drive (ukládání všech souborů, fotek a PDF) a Google Calendar API (synchronizace termínů). K autentizaci se používá Google Service Account.
* **Google Cloud AI:** * *Google Gemini Pro API* (Zpracování textu, RAG, generování reportů).
  * *Cloud Speech-to-Text* (Přepis diktovaných záznamů z terénu).
  * *Document AI / Cloud Vision* (OCR vytěžování lékařských a soudních zpráv).

## 4. Designová filozofie (UI/UX)

Uživatelské rozhraní musí striktně dodržovat následující designový jazyk:

1. **Aplikace Thinks jako vzor:** Extrémní minimalismus. Velké množství negativního prostoru (whitespace). Font *Inter*. Akcentní barvy se používají pouze pro stavy (červená = kritická událost, modrá = primární akce).
2. **In-place editace:** Zákaz používání modálních oken (pop-upů) pro běžnou editaci dat. Kliknutí na hodnotu ji změní na `<input>` přímo v řádku.
3. **Glanceable UI (Dashboardy):** Pro desktopové rozhraní (FosterFlow Organization) se využívají prvky z obchodních aplikací. Tabulky obsahují vysokou hustotu dat včetně *Sparklines* (miniaturních progress-barů přímo v řádku tabulky) pro okamžitý přehled čerpání limitů.
